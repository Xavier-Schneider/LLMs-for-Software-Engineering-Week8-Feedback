## 1. Counterexample Problems

### Problem A: Logging Hides a Race Condition

**Task Description:**
Debug this code. It is supposed to cache a user profile so that concurrent requests only fetch it once, but under concurrency it sometimes fetches the same profile multiple times. Identify the bug and fix it.

**Starter Code:**  
```python
import threading
import time
import random
import statistics

cache = {}
fetch_count = 0

def fetch_profile(user_id: str) -> dict:
    global fetch_count
    fetch_count += 1
    time.sleep(0.001 + random.random() * 0.001)  # fake I/O
    return {"user_id": user_id}

def get_profile_buggy(user_id: str) -> dict:
    # BUG: check-then-act with no lock
    if user_id not in cache:
        cache[user_id] = fetch_profile(user_id)
    return cache[user_id]

def run_trial(threads: int = 20) -> int:
    global cache, fetch_count
    cache = {}
    fetch_count = 0

    barrier = threading.Barrier(threads)

    def worker():
        barrier.wait()
        get_profile_buggy("alice")

    ts = [threading.Thread(target=worker) for _ in range(threads)]
    for t in ts:
        t.start()
    for t in ts:
        t.join()

    return fetch_count

results = [run_trial() for _ in range(20)]
print("Expected correct value: 1")
print("Observed fetch_count values:", results)
print("Average:", statistics.mean(results))
```

#### Guideline 3 Generate Debug Outputs:

**Guideline Definition**
For complex programs, first explain the bug to the model and ask it to add in debug outputs to the code so the problem can be analyzed deeper than plain code reading.

**Prompt and Context:**

**Solution or Output generated by GitHub Copilot:**  
```python
import threading
import time
import random
import statistics

cache = {}
fetch_count = 0
log_lock = threading.Lock()

def debug_log(msg: str) -> None:
    with log_lock:
        print(msg)
        time.sleep(0.005)  # simulate slow logging / console I/O

def fetch_profile(user_id: str) -> dict:
    global fetch_count
    fetch_count += 1
    time.sleep(0.001 + random.random() * 0.001)
    return {"user_id": user_id}

def get_profile_buggy(user_id: str) -> dict:
    debug_log(f"{threading.current_thread().name}: checking cache for {user_id}")

    if user_id not in cache:
        debug_log(f"{threading.current_thread().name}: cache miss for {user_id}")
        cache[user_id] = fetch_profile(user_id)

    debug_log(f"{threading.current_thread().name}: returning cached value")
    return cache[user_id]

def run_trial(threads: int = 20) -> int:
    global cache, fetch_count
    cache = {}
    fetch_count = 0

    barrier = threading.Barrier(threads)

    def worker():
        barrier.wait()
        get_profile_buggy("alice")

    ts = [threading.Thread(target=worker, name=f"T{i}") for i in range(threads)]
    for t in ts:
        t.start()
    for t in ts:
        t.join()

    return fetch_count

results = [run_trial() for _ in range(10)]
print("Expected correct value: 1")
print("Observed fetch_count values:", results)
print("Average:", statistics.mean(results))
```

**Analysis:**
The guideline assumes that adding debug outputs is a harmless way to gain more context. In this program, that assumption is false. The bug is a timing-sensitive race condition, and the added logging changes the thread schedule. Because `debug_log()` is slow and serialized with a lock, threads no longer hit the unsafe cache check at the same time. As a result, the race becomes much less visible, or even seems to disappear.

### 3. New and Updated Guidelines that worked for Problem A

#### Guideline 3: Classify the Bug
**Guideline Definition**
For complex programs, do not automatically add debug outputs first. First classify the bug. If it may be concurrency-sensitive, timing-sensitive, performance-sensitive, or memory-sensitive, avoid naive logging as the first step. Instead, prefer low-perturbation evidence such as minimal repro tests, assertions, counters, stack traces, debugger watchpoints, profilers, or race detectors. Add debug outputs only when they are unlikely to change the program's behavior, and keep them minimal and localized.

**Prompt and Context:**
Context: Above code

Prompt: Debug this code. It is supposed to cache a user profile so that concurrent requests only fetch it once, but under concurrency it sometimes fetches the same profile multiple times. Do not add debug prints first, because the bug may be timing-sensitive. Instead, create a minimal stress test, identify the root cause, and fix it with the smallest correct change. Then show that the fix works.

**Solution or Output generated by GitHub Copilot:**
```
import threading
import time
import random
import statistics

cache = {}
fetch_count = 0

def fetch_profile(user_id: str) -> dict:
    global fetch_count
    fetch_count += 1
    time.sleep(0.001 + random.random() * 0.001)  # simulate I/O
    return {"user_id": user_id}

def get_profile_buggy(user_id: str) -> dict:
    # BUG: unsynchronized check-then-act on shared state
    if user_id not in cache:
        cache[user_id] = fetch_profile(user_id)
    return cache[user_id]

cache_lock = threading.Lock()

def get_profile_fixed(user_id: str) -> dict:
    # Smallest correct fix: protect shared cache access
    with cache_lock:
        if user_id not in cache:
            cache[user_id] = fetch_profile(user_id)
        return cache[user_id]

def run_trial(fn, threads: int = 20) -> int:
    global cache, fetch_count
    cache = {}
    fetch_count = 0

    barrier = threading.Barrier(threads)

    def worker():
        barrier.wait()
        fn("alice")

    ts = [threading.Thread(target=worker) for _ in range(threads)]
    for t in ts:
        t.start()
    for t in ts:
        t.join()

    return fetch_count

def evaluate(fn, trials: int = 30):
    results = [run_trial(fn) for _ in range(trials)]
    return results, statistics.mean(results)

if __name__ == "__main__":
    buggy_results, buggy_avg = evaluate(get_profile_buggy)
    fixed_results, fixed_avg = evaluate(get_profile_fixed)

    print("Expected correct value per trial: 1")
    print()
    print("Buggy version:")
    print("  sample results:", buggy_results[:10])
    print("  average fetch_count:", buggy_avg)
    print()
    print("Fixed version:")
    print("  sample results:", fixed_results[:10])
    print("  average fetch_count:", fixed_avg)
```
**Analysis:**
The new guideline worked because it avoided adding logging to a timing-sensitive bug. Instead, it used a stress test to reproduce the issue, identified the unsynchronized access to cache, and fixed it by adding a lock. The same test then showed the fix worked: the buggy version fetched multiple times, while the fixed version fetched only once.
